El problema

Un sitio sin mantenimiento
pierde posición en silencio.

Los sitios web no son estáticos en su rendimiento aunque el código no cambie. Google actualiza constantemente cómo calcula los Core Web Vitals con datos reales de usuarios. Un score de 95 hoy puede ser 74 en seis meses sin que nadie tocara nada.

El schema puede entrar en conflicto con cambios en las especificaciones de schema.org. Los errores de cobertura en Search Console pasan desapercibidos. Las páginas dejan de indexarse. Y el médico no se entera hasta que nota que "ya no me llaman tanto".

El mantenimiento mensual es el protocolo que detecta eso antes de que impacte en los pacientes. No es gasto: es el costo de mantener el activo funcionando.

📉 Score que nadie monitoreó

El PageSpeed bajó de 91 a 63 en 4 meses. Causa: una imagen nueva subida sin comprimir. Nadie lo notó hasta que Search Console empezó a reportar CLS.

⚠️ Schema con error silencioso

Un campo requerido del schema Physician quedó vacío después de actualizar el horario. Google dejó de mostrar el rich snippet en los resultados. Pasaron 2 meses sin que el médico lo supiera.

🔍 Página de servicio desindexada

Un error en el sitemap generó que la página de "Consultas online" quedara excluida del índice de Google. 3 meses sin aparecer para esa búsqueda.

Protocolo mensual

Qué pasa exactamente
cada mes.

Cada plan tiene su protocolo. No hay sorpresas ni "depende". El día 1, el trabajo empieza. El día 5, el reporte está entregado.

  1. 📊

    Día 1 del mes

    Core Web Vitals + Search Console

    Ingreso a Search Console y reviso la sección Experiencia de página : estado de LCP, FID y CLS con datos reales de usuarios. Reviso también la cobertura: páginas excluidas, errores de indexación, advertencias de robots.txt. Todo se documenta con captura.

    Google Search Console Core Web Vitals
  2. 🔍

    Día 1–2

    Schema Validator + PageSpeed

    Corro el sitio por el Schema Markup Validator de schema.org . Si aparece algún error o warning, lo corrijo en el mismo ciclo. Luego ejecuto PageSpeed Insights en mobile y desktop y capturo el score. Si bajó de 90, entra al plan de corrección inmediata.

    Schema Markup Validator PageSpeed Insights
  3. 🔧

    Durante el mes

    Cambios menores + resolución de incidencias

    Hasta 1 hora de trabajo para cambios solicitados por el cliente: actualizar horarios, cambiar un texto, ajustar un formulario, corregir un link. Cualquier error técnico detectado durante el monitoreo se resuelve en el mismo mes, sin costo adicional.

    GitHub / FTP Trello / Notion
  4. 📦

    Antes del día 5

    Backup + reporte de entrega

    Hago backup del código en la carpeta del cliente en GitHub o Google Drive . Entrego el reporte mensual: capturas de PageSpeed, estado de Search Console, lista de tareas realizadas con fecha. No es un resumen vago: es evidencia.

    GitHub Google Drive

    Ubersuggest / Ahrefs AnswerThePublic
